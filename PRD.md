# PRD: SoundBatcher - 要件と将来構想

## 1. 目的

大量のサウンドファイルを高速にプレビューし、非破壊編集や一括処理（ノーマライズ、無音カット、リネーム）を直感的な操作で完結させるデスクトップアプリ。

## 2. ターゲットユーザー

* サウンドデザイナー、音楽プロデューサー
* ゲーム開発者（効果音アセット管理）
* ポッドキャスト編集者

## 3. 機能スコープ

### 3.1 現在実装済み（2026-02時点）

- ファイルブラウザ（Miller風2カラム、フォルダ移動、複数選択、ピン留め）
- タブ管理（追加、切替、並び替え、固定、他タブクローズ）
- コンテキストメニューによる操作（再生、キュー追加、パスコピー、フォルダ操作）
- ドラッグ&ドロップ（外部ファイル/フォルダの受け入れ、内部ドラッグでファイル移動）
- インスタント再生（ダブルクリック）と波形表示
- バッチキュー実行
- ラウドネス正規化
- リネームのみ実行モード
- 処理履歴（結果/エラー）の保存と閲覧
- ファイル一覧の自動更新（定期更新 + フォーカス復帰時）

### 3.2 今後の候補（Backlog / 構想）

### A. 超高速プレビュー

- **ファイルブラウザ**: フォルダツリー表示とドラッグ&ドロップ対応
- **インスタント再生強化**: スペースキー再生、カーソル移動連動再生などの操作最適化
- **リアルタイム波形表示**: `wavesurfer.js` 等を用いたシーク可能な波形ビジュアライザ

### B. スマート編集・一括処理

- **Auto Silence Truncation（無音除去）**: 閾値（dB）を下回る無音部を検知し、分割またはトリミング
- **Batch Normalization 拡張**: 最大振幅やLUFSなど、正規化手法の拡張
- **Smart Renamer 拡張**: 接頭辞/接尾辞/連番/日付などの高度な命名テンプレート
- **メタデータ編集**: ID3タグやWAVチャンク情報の簡易書き換え

### C. 書き出し（Export）

- 元ファイルを上書きせず、別フォルダに一括書き出し（非破壊編集）
- フォーマット変換（WAV, MP3, OGG, FLAC など）

## 4. 技術スタック (Recommended Stack for Cursor)

* **Framework:** [Tauri](https://tauri.app/) (軽量・高速・セキュア)
* **Frontend:** React + Tailwind CSS + Lucide Icons (UI)
* **Audio Library:**
  * Frontend: `wavesurfer.js` (ビジュアル用)
  * Backend (Rust): `symphonia` または `ffmpeg-next` (高速な音声解析・処理用)
* **State Management:** Zustand または Jotai

## 5. 設計品質要件（SOLIDベース）

### A. アーキテクチャ原則

* **単一責任（SRP）:** UI層、状態管理層、ドメインロジック層を分離する。
* **開放閉鎖（OCP）:** 新しい処理（例: ノイズ除去、EQ）を既存コード改修最小で追加可能にする。
* **依存逆転（DIP）:** ドメインロジックはFFmpeg実行やファイルI/Oの具体実装に直接依存しない。
* **インターフェース分離（ISP）:** 設定、履歴、処理結果の型をユースケース単位で分離する。
* **置換可能性（LSP）:** モック実装・本番実装を差し替えても同一の契約で動作する。

### B. 受け入れ基準（設計）

* 処理キューの同時実行ロジックは再利用可能な関数として分離されていること。
* タスク実行ロジックは、依存注入されたインターフェース経由で外部機能を利用すること。
* `App` コンポーネントはレイアウト統合を主責務とし、テーマ/言語反映は専用Hookで実施すること。
* ドキュメント（README/PRD）に設計原則と構成が明記されていること。

## 6. UI/UX デザイン案

* **3カラム構成:**
1. **左:** ファイルツリー / お気に入りフォルダ
2. **中央:** メイン波形表示 & 再生コントロール
3. **右:** インスペクタ 兼 一括処理パネル（リネーム、ゲイン調整等の設定）
   > 将来的には、パネルレイアウトをドラッグ&ドロップで編集できるようにしたい。
